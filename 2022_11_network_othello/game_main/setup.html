本実験はUnix系のOSで行うことを想定している． こちらではWindows/Macユーザ向けに自分のPC使用時の環境設定方法について述べる．ここに書いてあるのは唯一の方法というわけではなく，2022年9月時点での無難な方法という立場である．ここに記した方法以外にも，たとえば直にあるいはVirtualBoxやVMware Workstation Playerなどの仮想マシンモニタ上でLinuxやFree BSDをインストール・設定してもよい．

参照用プログラムの実行にはJava実行環境（JRE）が必要であるため，Javaの実行環境（JRE）をインストールする必要がある．

環境の整備：Windows (Cygwin; 簡単)

Cygwinを使うとWindows上に簡単にUnix環境を構築できる．ただ，一部の処理が遅かったり，特定のUnixをターゲットにしているようなコードは結局コンパイルできなかったりするので，その後より積極的にUnix環境を使いたいならばWSLを使うことを検討されたし．

手順1：Cygwinのインストール

Cygwin Installationのページから，setup-x86_64.exeを入手し，実行する．

インストール途中で，パッケージを選択する画面が出てくるので，gcc-core，make，doxygen，inetutilsを選択する． その後インストールを進め，完了させる．

インストールが成功すると，デスクトップにCygwin64 Terminalなるアイコンが表示される．

ヒント

  • Cygwin上のパスをWindows上のパスに変換するには，cygpathを使用する．たとえば，

    cygpath -w .

    を実行してみよう．

  • Cygwin上でカレントディレクトリでエクスプローラを使いたときは，

    cygstart .

    とする．大雑把にはcygstartは「開く」を実行するコマンドであり，引数のファイルの拡張子により適切なアプリケーションを起動する．

  • CドライブはCygwin上では/cygrive/c/としてアクセスできる．なので，たとえば，C:\Users\kztk はCygwin上だと/cygdrive/c/Users/kztk/としてアクセスできる．

  • インストールするパッケージを追加したり，アップデートしたりしたいときは，上記で入手したインストーラを実行する．

手順2：Java環境の準備

Microsoft Build of OpenJDKを使うのが楽であろう．

  1. 当該ページの「Download」をクリックし，遷移先のページからWindows用の.msiファイルをダウンロードする．
  2. ダウンロードした .msiファイルを実行し，インストールする．

環境の整備：Windows (WSL)

以下は最低限必要なもののみについて記述する．

本実験はunix環境を想定しているため，Windowsで実験を遂行する場合はWindows Subsystems for Linuxを用いるのが一つの方法である．

手順1：Ubuntuのインストール

コマンドラインを用いたインストール

比較的新しいWindows 10（version 2004以降）を利用しているのならば，PowerShellかコマンドプロンプトから

wsl --install

を実行すればよい．…とMicrosoftのサイトには書いてあるが私（松田）が2021年に試してみたところうまく行かなかったので，ひょっとしたら，

wsl --install -d Ubuntu

のようにディストリビューション名を指定する必要があるかもしれない．入ったLinuxがWSL 1で動くかWSL 2で動くかは

wsl -l -v

で確認できる（「VERSION」の列を見る）．ちなみに，インストールしたディストリビューションのWSLのバージョンの変更は

wsl --set-version Ubuntu 2

のようにすることで可能である（この例だとUbuntuのWSLのバージョンを2にする）．

旧来の手順

上記のコマンドラインからのインストールがうまく行かなければ，以下を実行する．

  1. 「設定」→「アプリ」→「アプリと機能」→「プログラムと機能」（右上）→「Windowsの機能の有効化または無効化」とすすみ，開いたウィンドウにて「Linux用Windowsサブシステム」にチェックをつけ「OK」を押す．

    • WSL 1でなくWSL 2を使用したいのならば「仮想マシンプラットフォーム」と「Windows ハイパーバイザー プラットフォーム」も有効にし， 下記の再起動後，PowerShellかコマンドプロンプトから

      wsl --set-default-version 2

      を実行する．

  2. Windowsの再起動が求められるので再起動する．
  3. 「Microsoft Store」（スタートメニューから見つかるはず）を起動し，”Ubuntu”を検索する．「Ubuntu」というのが見つかるはず
  4. 「入手」をクリックし，ダウンロードする
    • Microsoftアカウントが必要
  5. 「Ubuntu」を起動する（スタートメニューから見つかるはず）
    • ユーザ名とパスワードを適切に設定する
  6. これでUbuntuがインストールされる

WSL 2だと，特別な設定がなければ，Linux側はWindows側と同一のネットワークに接続されるわけではなく（つまり，ブリッジ接続はされない）， Windows側とのみ接続された仮想的なネットワーク上のマシンという扱いになる（一段階入れ子になる）． たとえば友人と同一ネットワークにWindowsマシンを接続して，さあ通信してみようとしてもWSL 2上のLinux側へ（Linux側からは問題ない）は（特別な設定なしには）接続できないことになる．

手順2：C環境（とJava実行環境）の準備

  • 以下のコマンドを実行する
sudo apt install gcc make doxygen default-jre

最初に，あるいはしばらくぶりにaptを実行する前には，sudo apt updateを実行するとよい．そうでなければパッケージが見付からない旨のエラーが出る場合がある．その後はsudo apt upgradeを行うとよい（特にsudo apt updateの後に実行が促される場合）．

手順4（オプショナル）：VSCodeのインストール

VSCodeを使うとWindows側からWSL側のファイルを編集できるので便利であろう．なお，VSCodeをインストールするのはWindows側であることに注意．

  1. VSCodeの公式ページからインストーラを入手し，その指示に従ってインストールする
  2. 拡張機能 Remote Development をインストールする．

起動する場合はWSL側から適当なフォルダの中で

code .

などとする．

環境の整備：Mac OS

同様に以下は最低限必要なもののみについて記述する．

手順1：Xcode command-line toolsのインストール

  1. XcodeをApp Storeから入手する（11 GB超あることに注意）
  2. 「ターミナル.app」（「アプリケーション」→「ユーティリティ」にある）を開き，以下のコマンドを実行する．
xcode-select --install

これで，makeやgccが使えるようになるはず（注意：ここでインストールされるgccは本物のgccではなく中身はclangである）．

https://brew.sh/ （日本語：https://brew.sh/index_ja）の指示にしたがって Homebrewをインストールする

手順3：telnetのインストール

以下のコマンドを実行する

brew install telnet

手順4：Java開発環境のインストール

HomebrewはJava実行環境のみのパッケージを提供していないので（2020年10月23日現在），Java開発環境をインストール必要がある．

以下のコマンドを実行する．

brew install openjdk

上記のコマンドの実行途中でopenjdkのインストール終了時点で下記のようなメッセージが表示される（あくまで以下は私の環境実行した場合のものである）

For the system Java wrappers to find this JDK, symlink it with
  sudo ln -sfn /usr/local/opt/openjdk/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk.jdk

openjdk is keg-only, which means it was not symlinked into /usr/local,
because it shadows the macOS `java` wrapper.

If you need to have openjdk first in your PATH run:
  echo 'export PATH="/usr/local/opt/openjdk/bin:$PATH"' >> ~/.zshrc

For compilers to find openjdk you may need to set:
  export CPPFLAGS="-I/usr/local/opt/openjdk/include"

なかでもとりわけ重要なのはPATHの設定である．もしzshを使っているのならば，表示されたメッセージにある通り，コマンド

echo 'export PATH="/usr/local/opt/openjdk/bin:$PATH"' >> ~/.zshrc

を実行する．終わったあとは

source ~/.zshrc

も実行する．

手順5：doxygenのインストール

以下のコマンドを実行する．

brew install doxygen
